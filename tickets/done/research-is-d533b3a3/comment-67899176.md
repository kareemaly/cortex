## Research Complete: OpenCode Default Prompt Accessibility

### Question
Is OpenCode's default provider prompt (`anthropic.txt`) accessible as a readable file from the installed npm package?

### Answer
**No.** The prompt is compiled into a Bun-native Mach-O binary at build time. The installed `opencode-ai` npm package contains only 6 files (wrapper script, platform binary, metadata). No `.txt` prompt files exist on disk.

### Key Findings
- `anthropic.txt` is imported via ES6 module syntax (`import ... from "*.txt"`) and inlined as a string constant by `Bun.build()` with `compile: true`
- The installed package is a thin npm wrapper that downloads a platform-specific native binary via `postinstall.mjs`
- The prompt IS extractable from the binary via `strings`, but this is too fragile for production use
- OpenCode v1.1.56 tested; binary format is Mach-O 64-bit arm64

### Recommendation
Use OpenCode's instruction file mechanism (Approach 4) — stop setting `agent.cortex.prompt` (which replaces the provider prompt) and instead pass Cortex's SYSTEM.md via instruction files. This preserves OpenCode's built-in `anthropic.txt` with zero drift.

Fallback: embed a static copy + fetch from GitHub during `cortex defaults upgrade`.
