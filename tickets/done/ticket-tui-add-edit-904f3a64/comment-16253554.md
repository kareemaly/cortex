Completed detailed implementation plan for both features (ticket TUI `e` shortcut and kanban popup).

Key design decisions:
1. **Editor (`e` key)**: New daemon API endpoint `POST /tickets/{id}/edit` -- follows existing `ExecuteCommentAction` pattern where daemon resolves file paths and calls `TmuxManager.DisplayPopup()`. Added `IndexPath()` method to ticket Store.
2. **Kanban popup (`o`/`Enter`)**: New daemon API endpoint `POST /tickets/{id}/show-popup` -- runs `cortex show <id>` in a tmux popup. Fully removes the embedded inline ticket detail view from kanban.
3. Both features use the non-blocking `DisplayPopup` via daemon HTTP endpoints, consistent with the architecture.
