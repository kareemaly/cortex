## Research Complete: Claude Code vs OpenCode Agent Defaults

### Key Finding
All 16 prompt files are **identical** between the two agent types. The differences are entirely in configuration and runtime behavior.

### Major Differences

1. **cortex.yaml args** — Claude Code has carefully scoped CLI args per session type (permission mode, tool allowlists). OpenCode has **zero args** configured for any role.

2. **Lifecycle hooks** — Claude Code gets 3 hooks (PostToolUse, Stop, PermissionRequest) that enable real-time agent status in the TUI. OpenCode has none — explicitly skipped in spawn code.

3. **Permission model** — Claude Code uses `--allowedTools` to pre-approve specific MCP tools per role. OpenCode runs with blanket `bypassPermissions` + `"*": "allow"`.

4. **Runtime spawning** — Different binaries, different config delivery (files vs env var), different prompt injection mechanisms. OpenCode config is generated by transforming Claude-format MCP config at runtime.

### Gaps in OpenCode
- No CLI args (no permission scoping, no plan mode)
- No lifecycle hooks (no TUI status tracking)
- No session resume support
- More permissive security posture
