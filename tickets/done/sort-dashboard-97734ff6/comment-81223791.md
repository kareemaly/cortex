## Sort Dashboard Sessions by Most Recent First

### Changes

**`internal/types/response.go`** — Added `SessionStartedAt *time.Time` field to `TicketSummary` struct. Pointer type so it's omitted from JSON when no active session exists.

**`internal/types/convert.go`** — Populated `SessionStartedAt` from `sess.StartedAt` inside the existing `if sess != nil` block in `ToTicketSummary`.

**`internal/cli/tui/dashboard/model.go`** — Three changes:
1. **Duration display** (`renderSessionRow`): Uses `SessionStartedAt` when available instead of `ticket.Updated`, showing time since session start rather than last ticket update.
2. **Session row sorting** (`rebuildRows`): Collects all active-session tickets from both Progress and Review into a single slice, sorts by `SessionStartedAt` descending (most recent first), then emits rows.
3. **Active project sorting** (`rebuildRows`): When both projects are active, sorts by newest session time descending. Added `newestSessionTime` helper that checks both architect session and ticket sessions.
