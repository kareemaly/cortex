## Root Cause

Two issues:

1. Commit 847e3da removed the `cortex show` CLI command but left 4 stale references in daemon/spawn/SDK code
2. The replacement subcommand `ticketShowCmd` had `Use: "<id>"` — Cobra treats this as a literal command name, so `cortex ticket show <uuid>` (or `cortex ticket <uuid>`) could never route to it

## Fix (5 files, single commit)

- **`cmd/cortex/commands/ticket_show.go`** — Changed `Use: "<id>"` to `Use: "show <id>"` so Cobra routes `cortex ticket show <id>` correctly
- **`cmd/cortex/commands/ticket.go`** — Reverted to simple group command (no positional arg delegation)
- **`internal/daemon/api/tickets.go`** — Updated popup command to `cortex ticket show %s`
- **`internal/core/spawn/spawn.go`** (2 locations) — Updated companion cmd to `cortex ticket show %s` with explicit ticket ID arg
- **`internal/cli/sdk/client.go`** — Updated comment
